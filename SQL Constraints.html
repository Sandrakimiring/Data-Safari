SQL Constraints

Constraints in SQL are rules enforced on data in tables to maintain accuracy and integrity.

1. NOT NULL

Ensures that a column cannot have NULL values.

CREATE TABLE employees (
    id INT NOT NULL,
    name VARCHAR(100) NOT NULL
);

2. UNIQUE

Ensures all values in a column are unique.

CREATE TABLE employees (
    email VARCHAR(100) UNIQUE
);

3. PRIMARY KEY

A combination of NOT NULL and UNIQUE. It uniquely identifies each row.

CREATE TABLE employees (
    id INT PRIMARY KEY,
    name VARCHAR(100)
);

4. FOREIGN KEY

Links two tables together, enforcing referential integrity.

CREATE TABLE orders (
    order_id INT PRIMARY KEY,
    customer_id INT,
    FOREIGN KEY (customer_id) REFERENCES customers(id)
);

5. CHECK

Ensures that all values in a column satisfy a specific condition.

CREATE TABLE employees (
    age INT CHECK (age >= 18)
);

6. DEFAULT

Sets a default value for a column if no value is provided.

CREATE TABLE employees (
    salary DECIMAL(10,2) DEFAULT 50000.00
);
