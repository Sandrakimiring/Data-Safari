Introduction

What is SQL?

SQL (Structured Query Language) is a standard language designed for managing and manipulating data in relational databases. It allows users to store, retrieve, update, and delete data efficiently in systems like MySQL, PostgreSQL, SQLite, and SQL Server.

Why Learn SQL?

SQL is a foundational skill for anyone working with data:

  • Data Analysis: Query databases to extract insights for decision-making.
  • Database Management: Create and maintain structured data storage.
  • Backend Development: Power web applications with dynamic data.
  • Big Data: Process large datasets in tools like Hadoop or Spark SQL.
  • Business Intelligence: Generate reports and dashboards.

Learning Outcomes

By mastering SQL, you’ll be able to:

  • Design and manage relational databases.
  • Write queries to retrieve and analyze data.
  • Optimize database performance for large-scale applications.
  • Integrate SQL with programming languages like Python.

Basic Commands

Overview

Basic SQL commands allow you to create, modify, and query databases. Here are the essentials:

1. Creating a Table

CREATE TABLE students ( id INT PRIMARY KEY, name VARCHAR(50), age INT, grade CHAR(1) );

2. Inserting Data

INSERT INTO students (id, name, age, grade) VALUES (1, 'Alice', 20, 'A');

3. Selecting Data

SELECT * FROM students;

4. Updating Data

UPDATE students SET grade = 'B' WHERE name = 'Alice';

5. Deleting Data

DELETE FROM students WHERE id = 1;

Learning Outcomes

After this section, you’ll be able to:

  • Create and populate database tables.
  • Retrieve and modify data with basic queries.

Advanced Queries

Overview

Advanced queries allow for more complex data retrieval and analysis:

1. Filtering with WHERE

SELECT name, age FROM students WHERE age > 18;

2. Sorting with ORDER BY

SELECT * FROM students ORDER BY age DESC;

3. Aggregating with GROUP BY

SELECT grade, COUNT(*) as count FROM students GROUP BY grade;

Learning Outcomes

You’ll learn to:

  • Filter and sort data effectively.
  • Perform aggregations like counts, averages, and sums.

Joins

What are Joins?

Joins combine data from multiple tables based on related columns.

1. INNER JOIN

SELECT students.name, courses.course_name FROM students INNER JOIN courses ON students.id = courses.student_id;

2. LEFT JOIN

SELECT students.name, courses.course_name FROM students LEFT JOIN courses ON students.id = courses.student_id;

Learning Outcomes

You’ll be able to:

  • Link tables to retrieve combined data.
  • Understand different join types (INNER, LEFT, RIGHT, FULL).

Indexes & Optimization

What are Indexes?

Indexes improve query performance by allowing faster data retrieval.

Creating an Index

CREATE INDEX idx_name ON students(name);

Optimization Tips

  • Use indexes on frequently queried columns.
  • Avoid over-indexing to reduce write overhead.
  • Analyze queries with EXPLAIN to identify bottlenecks.

Learning Outcomes
